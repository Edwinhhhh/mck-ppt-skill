# McKinsey 风格 PPT 大纲生成 Skill

## 什么时候使用

当用户希望将以下内容转化为适合管理层沟通、项目汇报、董事会讨论或咨询式表达的 PPT 大纲时，使用本 Skill：

- 长文章或研究材料
- 内部备忘录
- 会议纪要
- 商业问题定义
- 项目复盘材料
- 多段零散业务素材
- 已有初步结论但缺少清晰页面结构的汇报内容

如果用户的目标是生成完整演讲稿、营销文案、海报文案或视觉设计稿，则不应直接套用本 Skill 的输出方式。

## Skill 定位

本 Skill 的目标不是生成“好看但空泛”的 PPT 文案，而是生成可直接用于汇报落稿的咨询式页面大纲。

输出应体现以下特征：

- 面向管理层阅读
- 结论先行
- 逻辑清晰
- 单页单信息
- 可继续扩展为正式幻灯片
- 每页有明确的图表或视觉表达建议

## 工作原则

### 1. Conclusion-first

先给整体结论，再展开依据、支撑和建议。

要求：

- 开头先给 overall takeaway 或核心判断
- 每页标题尽量写成完整观点句
- 避免把页面标题写成宽泛名词，例如“市场分析”“项目进展”“用户洞察”

### 2. One-slide-one-message

每页只表达一个核心信息。

要求：

- 每页必须有唯一主旨
- 支撑要点服务于该页主旨，不引入无关信息
- 如果一个主题包含多个并列结论，应拆成多页

### 3. Storyline-driven

整份大纲需要具备从问题到判断、从判断到建议的推进结构。

常见 storyline 结构包括但不限于：

- Context → Insight → Implication → Action
- Problem → Cause → Option → Recommendation
- Where we are → Why it matters → What to do next
- Situation → Complication → Resolution

### 4. Executive tone

语言保持专业、克制、判断清晰，适合管理层阅读。

要求：

- 避免宣传口吻、情绪化措辞和过度修饰
- 尽量使用判断句而非描述性空话
- 优先表达业务含义，而不是堆砌形容词

### 5. Chart suggestions

为每页提供最合适的图表或视觉表达建议。

要求：

- 图表建议服务于观点表达
- 指出推荐图表类型，以及该图表要表达的重点
- 若更适合使用表格、流程图、时间线、2x2 矩阵，也应明确说明

## 常见输入类型

本 Skill 常见输入包括：

- 文章原文
- 会议纪要全文
- 备忘录或邮件内容
- 商业议题描述
- 项目背景、问题与目标
- 调研摘要
- 多段零散素材与观察点
- 已有结论、假设与若干数据点

如果输入较零散，应先整理信息，再生成大纲。

## 输入处理方式

### 1. 先识别任务目标

优先判断以下信息：

- 汇报对象是谁
- 这份材料要回答什么核心问题
- 输出偏战略、运营、市场还是财务
- 期望页数范围是多少
- 用户是否已有预设结论或判断方向

### 2. 提炼有效信息

从原始材料中优先提取：

1. 关键结论
2. 核心事实与数据
3. 主要问题或矛盾
4. 支撑性观察
5. 建议动作或待决策事项

### 3. 处理信息不足

如果原始信息不足以支撑完整判断：

- 不编造数据
- 不伪造事实
- 显式标注 assumptions
- 明确指出仍需补充的数据或信息

### 4. 处理零散素材

当输入是碎片化内容时，可先在内部按以下方式整理：

- Facts：事实与数据
- Insights：判断与洞察
- Questions：待回答问题
- Actions：建议动作

再据此构建 storyline。

## 输出结构要求

默认输出应包含以下部分：

### 1. Overall takeaway

用 1-3 句话概括整份材料的核心结论或总论。

### 2. Storyline

用简洁列表给出整份 deck 的逻辑主线，通常对应 4-8 页的页面推进顺序。

### 3. Page-by-page outline

逐页输出大纲。每页至少包含：

- Slide title
- Key message
- Supporting points
- Chart suggestion

其中：

- Slide title：必须尽量写成观点句
- Key message：用 1-2 句话说明本页核心信息
- Supporting points：列出 3-5 个支撑要点
- Chart suggestion：说明推荐图表或视觉表达方式

## 风格约束
输出时应遵循以下约束：
- 中文为主，允许少量必要英文术语
- 风格专业、简洁、清晰
- 不写成营销文案或演讲稿
- 不使用空泛口号式表达
- 不堆砌“非常重要”“显著提升”“强势增长”等无依据措辞
- 每页标题尽量具备判断性
- 若信息不充分，明确指出 assumptions 与待补充信息
## 常见任务类型
本 Skill 适合处理以下任务：
### 1. 文章转 PPT 大纲
将一篇长文提炼为适合高层汇报的页面结构。
### 2. 会议纪要转管理汇报
将讨论记录整理为结论、问题、建议与下一步动作。
### 3. 商业主题直接展开
针对某个业务问题，生成一套咨询式 storyline 与页面大纲。
### 4. 零散素材整合
将多个碎片化输入整理为可汇报的结构化 deck outline。
### 5. 备忘录改写
将内部 memo 改写为更适合高管阅读的 PPT 页面逻辑。
## 额外要求
在生成结果时，优先做到以下几点：
- 标题可直接作为页面标题使用
- 每页内容适合后续落成真实幻灯片
- 页面之间有明确逻辑递进关系
- 图表建议与页面结论匹配
- 若用户指定页数，尽量严格控制在指定范围内
## 不建议的输出方式
避免以下情况：
- 只有摘要，没有页级结构
- 标题全是名词短语，没有结论
- 一页中混入多个无关主题
- 输出成逐字演讲稿
- 编造数据、案例或结论
- 使用过强营销腔或广告语气
## 默认执行要求
在未被用户特别覆盖时，默认按以下标准执行：
- 输出咨询式 PPT 大纲，而非全文改写
- 优先生成 5-8 页的 page-by-page outline
- 标题使用观点句
- 每页给出 chart suggestion
- 保持 executive tone
- 对信息不足部分显式标注 assumptions
